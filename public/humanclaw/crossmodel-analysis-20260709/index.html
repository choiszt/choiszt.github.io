HCBv6:16 模型跨模型误差分析(Cross-Model Error Analysis v1)

日期:2026-07-09 · 数据:15 VLM 全量 rollouts 包(20260705)+ GPT-5 本地 run · 脚本:outputs/gdrive_dl/(t1_stream / crossmodel_t2) 姊妹篇:GPT-5 单模型深度误差分析(step 级机理最深),本报告回答"所有模型如何失败、什么是任务共性、什么是家族/模型特有"。

口径说明:16 模型 = 大包 15 个 VLM + 本地 GPT-5 (non-thinking)。统一成功判据(主动 Stop 且身体-目标 <0.2m)、统一 step 级抽取管线、统一 taxonomy。已一致剔除出生下陷缺陷场景 102343992(16 模型全部中招,见 §6),n=1188/模型。所有 run 均为 07-07 管线修复之前的代码,系统性问题见 §6 的不均匀污染分析。

0. TL;DR

排行:Gemini 3.1(43.4%)>> Gemma 4(29.4%)> Gemini 2.5(22.1%)≈ Qwen3.6-27B(21.4%)> GPT-5(14.4%)≈ Qwen3.5-27B(13.7%*)> … > 尾部 <1%。

三个共性问题(跨家族普遍): 1. 终止召回缺失(接触目标却不 Stop):除秒死模型外全员存在,占失败 6–28%——高层决策层是全行业软肋 2. 长路径导航衰减:所有模型的失败主体仍是"到不了"(导航层占失败 73–91%) 3. 下陷场景全灭:16/16 模型在 102343992 碰撞率≈100%、成功≈0——环境缺陷的完美对照实验

最重要的方法论发现——系统污染不均匀:verifier id/name bug(我们在 GPT-5 分析中定位的解析缺陷)对不同模型的杀伤天差地别:qwen35_27b 的 71% episode 被它杀死(990/1032 次停止是 bug),qwen36_35b_a3b 42.6%、qwen35_122b 22%、GPT-5 16%,而 Gemini/Claude/Gemma 完全免疫(0 例)。跨模型排行如果不做 bug 校正,对 Qwen 家族系统性不公平。

家族/模型特有问题:Claude 4.8 = 重度"看见走不到"(claim 100% 但 44% 失败于接近不了)+ 停止精度全场最低档(24.5%);Qwen-A3B/小模型 = 高破坏性(每集撞倒 1.2–1.3 个物体);deepseek_ocr = 98.8% 一步秒死(管线不兼容);gemma4 = interact 指标层缺失(数据问题)。

1. 数据与方法

  • 数据:hcbv6_completed_15vlm_full_rollouts_20260705.tar.zst(114.7GB,18266 rollouts,65 run roots)+ GPT-5 本地 run(1202 集)。
  • 管线:单遍流式解析全部 episode_log(136 万 step),抽取每步的目标距离(身体 56 关节到最近实例 AABB)、位置、动作、感知声明(alias+negation 匹配 visible_state)、停止事件与归因字段(verifier verdict / proposed / final_action_name)、fallback 标记;与 metrics.json(成功/坐/撞物)按 episode 连接。
  • 统一 taxonomy(每个失败 episode 唯一归类,优先级从上到下):
类别 定义 层
system_stop 停止由 verifier id/name 错配或 planner fallback 产生(非模型意图) 系统
model_wrong_stop 模型自主提议 Stop 但停在 >0.2m 决策(精度)
contact_no_stop 曾接触 ≤0.2m 但从未有效 Stop 决策(召回)
other_stop_fail 其他停止失败(格式异常等) 系统/混合
last_mile 到过 ≤1m,拿不下 0.2m 接触 近距控制
claimed_not_approached 声称看见但从未到 1m 导航
never_found 从未声称看见 感知/探索
  • 可信度注记:闭源模型(Gemini/Claude)与 gemma4 的 raw 输出格式与 Qwen/GPT 管线不同,其停止事件多归入"模型自主"侧的粗桶(见 §6.3);gemma4 的 interact 层字段缺失;deepseek_ocr 每集 1 步,行为指标无意义。

3. 错误类别体系与逐类分析

类别 I:感知/探索失败(never_found)

  • 重灾:internvl(26.9%)、qwen3_omni(22.2%)、qwen3vl_8b(16.7%)、qwen35_35b(17.6%)
  • 头部模型 5–12%;claude48 为 0%(它什么都"声称看见",见类别 VI 注)
  • 共性形态(与 GPT 深度分析一致):长路径 episode 上探索绕圈,claim 匹配器无正面提及

类别 II:导航执行失败(claimed_not_approached)——多数模型的最大失败类

  • 除被 bug 屠杀的 qwen35_27b 外,几乎所有模型的第一大失败类(16–48%)
  • 极端:qwen3vl_30b 48.0%、claude48 44.4%、qwen3_omni 33.8%、GPT-5 30.4%
  • Claude 的形态特殊:claim 率 100%(每集都声称看见)但 1m 到达率仅 32%——"什么都说看见 + 走不过去",疑似把远处/隔墙目标当可达,或声明泛滥
  • 共性结论:看见≠能走到,跨模型的 claim→1m 转化率仅 32–66%,长程导航是任务的第一瓶颈(与 GPT 报告的难度梯度分析互证)

类别 III:最后一米失败(last_mile)

  • 普遍 2–15%;小模型更重(qwen3vl 系 15%)
  • GPT 深度分析给出的机理(近距离深度误判、开环微调、tv 判据物理性)预计跨模型成立;tv 类别的物理可达性问题对所有模型一视同仁

类别 IV:高层决策失败——"低层对、高层错"(paper 重点)

两个子类,方向相反:

IV-a 终止召回缺失(contact_no_stop)——最强共性问题 - 全员存在(秒死模型除外):qwen3vl_8b 28.2%、qwen35_35b 24.7%、internvl 23.1%、qwen35_122b 19.8%、qwen36_27b 19.1%、GPT-5 18.3%、claude 10.2%、gemma4 9.8%、gemini31 3.5% - 摸到了目标却不敢/不会做出"宣布完成"的不可逆决策——与 GPT-5 报告的"precision 高 recall 低"结论在 15 个模型上复现,且与模型档次强相关(gemini31 几乎没有)

IV-b 停止精度不足(model_wrong_stop) - 模型自主停止的落点精度(P(≤0.2m | 自主 stop),仅对可归因模型):gemini31 61%、gemma4 68%、qwen36_27b 69% vs claude48 24.5%、qwen3_omni 24.3%、gemini25 40.6% - gemini25/gemini31/claude 的"错停"占失败 27–29%:它们敢停,但相当比例停错位置(深度/完成度误判) - 与 IV-a 合看:强模型的问题是"停得多而精度有限",弱模型的问题是"根本不敢停"——终止决策的两种病理

IV-c sit-interact(bed/couch/toilet,接触后) - 接触后真的去坐的比例:gemini31 93%、gemini25 86%、gemma4 77% vs qwen36_27b 57%、qwen35_122b 28%、qwen35_27b 26% - 三种失败:不知道要坐(internvl 类)/坐了不成(qwen25vl)/坐了但旧协议不计分(全员,协议问题)

4. 系统污染专节:不均匀的 bug 杀伤(方法论核心)

我们在 GPT-5 深度分析中定位的 verifier id/name 错配 bug(verifier 文本与 final_action_name 写 Turn,final_action_id 误填 1=Stop,代码按 id 解析→episode 被杀)在 15 个模型上的触发率极不均匀:

模型 id-bug 停止数 占全部停止 占全部 episode 其他系统停止
qwen35_27b 990 96% 71.0% ⚠️ —
qwen36_35b_a3b 570 99% 42.6% —
qwen35_122b 360 86% 22.0% —
qwen36_27b 303 67% 12.7% —
gpt5(本地已归因) 193 49% 16.0% —
qwen25vl_32b 0 — — fallback 180 + verifier 显式停 268(空/异常输出)
gemini31 / gemini25 / claude48 / gemma4 ≈0 — ≤0.3% —

解读: 1. bug 的触发取决于"该模型担任 verifier 时是否写出 id/name 不一致的 JSON"——Qwen 家族高发,Gemini/Claude/Gemma 几乎不犯。同一个管线缺陷,对不同模型是 0 到 71% 的天壤之别。 2. qwen35_27b 的 13.7% 严重失真:它 71% 的 episode 在中位第 24 步就被杀;幸存 episode 中接触→成功转化率反而很高(sit 后成功 96%)。若按幸存集成功率外推,其真实水平可能在 30–47% 区间(上界外推激进,须重跑确认)——从第 6 名跳到潜在第 2 名的量级。 3. qwen36_35b_a3b(5.9%)、qwen35_122b(10.9%)、qwen36_27b(21.4%)同理被低估。 4. paper 处理建议(按优先级):①用修复后管线(07-07 四修复)重跑 Qwen 系受害模型的被杀 episode(qwen35_27b 843 集、qwen36_35b_a3b 506 集…);②至少在主表标注每模型 system_stop 率并给 bug-adjusted 区间;③绝不可用当前原始分数直接比较 Qwen vs 其他家族。

其他系统类:qwen25vl 的空输出链(fallback+显式停,合计 35.4% episode)、deepseek_ocr 一步秒死(输出完全不可解析,98.8%)、gemma4 interact 指标缺失(其 run 用了缺模块的 runner)、出生下陷场景(16 模型全灭,已统一剔除)。

5. 共性问题 vs 家族/模型特有问题(总结矩阵)

任务共性(所有模型,能力无关): | 问题 | 证据 | |---|---| | 长程导航是第一瓶颈 | 失败中导航层占 73–91%(全员);claim→1m 转化 32–66% | | 终止召回缺失 | contact_no_stop 全员 3.5–28%,与档次负相关 | | 下陷场景全灭 | 16/16 碰撞率≈100%、成功≈0(环境缺陷) | | tv 类物理判据难 | 各模型 tv 成功率一致垫底(见分类别数据) |

家族特征: | 家族 | 特征 | |---|---| | Gemini | 敢停、多停(错停占失败 27–29%)但整体转化最高;运动最文明 | | Qwen(3.5/3.6) | verifier id-bug 高发(12–71% episode 被杀);A3B/小模型破坏性高、终止召回差 | | Claude | "全声称看见 + 走不到"(claim 100%、到 1m 仅 32%);停止精度最低档(24.5%) | | 开源小模型(≤8B/A3B) | 接触率尚可(19–29%)但接触→成功≈0;"腿好脑坏"的极端形态 |

6. 对 paper 的建议

图表: 1. 主表:16 模型成功率 + system_stop 率 + bug-adjusted 区间(诚实呈现污染) 2. 堆叠条形:统一 taxonomy 分布 ×16(§2.1 的可视化)——"每个模型怎么死"一图流 3. 漏斗折线:claim→1m→contact→success ×16(§2.2)——"死在哪一关" 4. 散点:接触率 vs 接触→成功转化率(x=运动层,y=决策层)——四象限直观呈现"腿好脑坏" 5. Disturbance 副表:每集撞倒物体 vs 成功率

可直接下的 claim: - 终止决策(何时宣布完成)是跨模型共性软肋,且强弱模型呈现两种相反病理(不敢停 vs 停不准) - 长程导航衰减普遍且陡峭,感知不是主要瓶颈(claim 率普遍 >75%) - 评测分数对 scaffold 解析细节高度敏感,且敏感度因模型而异(0–71%)——agentic benchmark 方法论的重要教训 - 运动质量(破坏性)与任务能力正相关

必做修正(发表前): 1. Qwen 系受害 episode 用修复管线重跑(约 2000 集,~1 天) 2. deepseek_ocr/gemma4 的数据问题处理(脚注/剔除) 3. 下陷场景统一剔除口径写入 setup
