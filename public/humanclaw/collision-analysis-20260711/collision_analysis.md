# HCBv6 碰撞溯源分析：碰撞是怎么来的（2026-07-11）

> 步级接触记录逐条归因：撞的是什么、离目标多远、执行什么技能、哪个身体部位、持续多久。
> 数据 = 16 模型 × ~1,188 集（剔除下陷场景 102343992 与 init-stuck 出生卡死集）× 每步接触列表，约 136 万步。
> 论文引用口径：9 个入表模型。

## TL;DR

1. **Coll% 不能直接读作 "body misawareness" 的强度**。碰撞步的构成（9 模型均值）：
   **目标接触 49%**（任务固有：走到跟前/坐/贴着微调）＞ **迎面撞上 24%**（障碍在正前方视野内、前进中撞上）＞ **途中擦碰 20%**（移动/转身中 1–2 步轻擦）＞ **卡住硬顶 4%**（贴着家具 ≥5 步硬推不动）＞ **落位失误 3%**（≤1m 收尾动作撞到非目标）。
2. **地毯是测量噪声，不是行为类别**：脚踩 rug/carpet/mat 被计为"非地面碰撞"（rug 是独立 rigid object），占接触步 12–19%（事件级 3–8%）。从 Coll% 剔除后每模型降 5.7–8.0 个点、**排序完全不变**。建议从指标里剔除并在 caption 注明。
3. **类别普适、幅度是签名**：五类在所有 9 个模型上都非零（统一趋势成立）；模型差异体现在幅度——Claude-4.8 迎面撞上 38.7%（全场最高，和它 100% claim 的过度自信画像自洽）；GPT-5 (non-thinking) 卡住硬顶 9.0%（全场最高，和它"不终止原地耗尽"自洽）；表外小模型（qwen3vl-30B / qwen-omni）wedged+frontal 合计 >50%。
4. **"真正的墙"不在数据里**：论文碰撞口径把 stage（含墙体/地板）整体排除，"卡进墙"在此表现为卡进大件家具/柜体。
5. **落位失误在碰撞里只占 2–4%，但这不代表交互失败少**——"坐进空气"（61% 的 sit 失败）是零接触的，碰撞数据天然看不到；要与 sit 管线的分析拼合才是完整图景。

## 1. 方法

**事件化**：每个 episode 内连续接触的步合并为一个"碰撞事件"，按持续步数加权（卡死 30 步 ≠ 30 次独立碰撞，而是一个重事件）。

**两层分类**：
- 测量层：整个事件只碰地毯/垫子（main_category ∈ {carpet, rug, mat, doormat} 或 super_category=floor_covering）→ `artifact_floorcover`，剔出行为分母，单独报告。
- 行为层判定树（优先级从上到下）：
  | 类 | 判据 |
  |---|---|
  | ① target_contact 目标接触 | 事件内 ≥50% 步接触目标（接触点落在目标 AABB+10cm 内） |
  | ② wedged_stuck 卡住硬顶 | 非目标接触持续 ≥5 步且位移 <5cm/步 |
  | ③ lastmile_misplace 落位失误 | 中位距离 ≤1m 且事件多数步为收尾技能（sit/侧移/转身/后退/站立） |
  | ④ frontal_walkin 迎面撞上 | walk_forward 前进中（位移 ≥5cm），最强接触方向与运动方向夹角 <60° |
  | ⑤ enroute_brush 途中擦碰 | 其余（多为转身/行进中 1–2 步轻擦，中位长度 2 步） |

**口径注意**：
- 每步取最后一个物理帧的接触列表采样（1/15 帧）；绝对接触率略低估，构成比例无偏。
- 论文 Coll%（from_logs 管线）排除全部 stage 几何（含墙体），只计与 object 的接触；本分析与其一致。
- 被撞对象经 HSSD 语义表（objects.csv，28 位 hash 前缀匹配）解析；约 25% 的接触解析为 unknown（疑似门等 articulated 构件，语义表未收录）。
- 曾评估第六类"原地转向扫碰"（rotation_scrape：≥50% turn 且位移 <5cm）：仅占 0.4–1.2%，不足以独立成类，并回 ⑤。

**统一趋势（所有模型共享）**：
- 最大成分永远是目标接触（42–60%）——Coll% 的一半左右是"到了目标跟前"的任务固有接触，且被不终止行为（贴着目标耗步）放大。
- 第二大失误型成分是迎面撞上（14–39%）——障碍应在视野正前方仍然撞上，直接的 ego 空间感知失败。
- 强模型的残余碰撞偏"轻擦"（Gemini 双子 ~30% 途中擦碰、1–2 步即过），弱模型偏"重撞"（迎面+卡住），碰撞的"质"比"量"更能区分模型。

**模型签名（幅度差异）**：Claude-4.8 迎面 38.7%（其余 ≤30）；GPT-5 卡住 9.0%（其余 ≤8.4，多数 ≤4）；InternVL 卡住 8.4% + 迎面 25.6%。

## 4. 地毯到底算不算一类？

**结论：不算行为类别，归测量层 artifact。** 理由：
1. 行为五类回答"为什么撞"，地毯不是行为原因——脚踩静态 rug 是正常行走，无物理后果（会被踢走的地毯已计入 #Dtb）。
2. 模拟器内置 static collision metric 本来就过滤"低矮薄面接触"（thin<5cm / 低于 8cm 忽略）；论文 from_logs 管线不过滤反而不一致。
3. 定量上无害但有噪声：剔除后排序不变，语义更干净。
4. 处理建议（二选一）：(a) 主表 Coll% 定义加 floor-covering 过滤并重跑；(b) 主表不动，error analysis 用 no-rug 口径 + caption 一句话说明。

## 6. 对 paper 的建议

1. Coll% caption 补充定义："fraction of executed steps with any non-ground body contact"；建议剔除 floor coverings 并注明（或至少脚注说明其占比 12–19%、跨模型均匀）。
2. Error analysis 可引用的两句话结论：(a) 碰撞的一半是任务固有的目标接触，失误型碰撞以"视野内迎面撞上"为主（24%）——ego 空间感知而非运动控制的失败；(b) "巡航撞家具率"（>2m 接触非目标物体）是比 Coll% 干净的 body-awareness 指标，强弱模型差 2.5 倍。
3. 与 body-awareness 叙事的衔接：碰撞普遍存在且模型全程无感（含成功集）+ 贴着目标不知道到了（contact_no_stop）是 body-misawareness 的干净证据；turn-deadlock/spatial trap 不是（死锁集碰撞率不高于成功集）。
